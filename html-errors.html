10 Common HTML Mistakes And Correct Usage

Frequently Observed HTML Errors with Correct Usage
HTML Tag / Concept Explanation Wrong Example Correct Example
<input> missing name Form inputs must have a name attribute for data submission. <input type="text"> <input type="text" name="username">
<iframe> missing title Iframes must include a title for accessibility. <iframe src="video.html"></iframe> <iframe src="video.html" title="Intro video"></iframe>
Using <b> for emphasis Use semantic tags like <strong> instead of <b> for meaning. <p>This is <b>important</b></p> <p>This is <strong>important</strong></p>
<ul> / <ol> Lists must contain only list items. <ul><p>Item</p></ul> <ul><li>Item</li></ul>
<a> Links must have meaningful text. <a href="page.html">Click here</a> <a href="page.html">View contact page</a>
<table> Tables must include headers for accessibility. <table><tr><td>Name</td></tr></table> <table><tr><th scope="col">Name</th></tr></table>
<form> Forms should group related controls. <form><input><input></form> <form><fieldset><input><input></fieldset></form>
<button> Buttons must specify their type. <button>Submit</button> <button type="submit">Submit</button>
<lang> Document language must be declared. <html> <html lang="en">
<pre> Code must be escaped inside preformatted text. <pre><p>Text</p></pre> <pre><code>&lt;p&gt;Text&lt;/p&gt;</code></pre>
